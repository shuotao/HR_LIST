### 對齊的概念
- **架構選擇：Firebase Hosting + Cloud Run（非 Cloud Functions）**：因為 MarkItDown + 依賴超過 500MB，Cloud Functions gen2 有部署大小限制，Cloud Run 無此限制且同樣有 2M 請求/月免費額度
- **移植而非引用**：評分邏輯和擷取邏輯是「複製移植」到 `web/backend/` 的獨立檔案，不是 import 原始腳本。原因：(1) 不修改原始專案檔案的約束 (2) Cloud Run 容器無法存取本機 `.agent/skills/` 路徑
- **Blaze 方案為必要條件**：Cloud Run 必須升級 Firebase 為 Blaze（按量付費），Spark 免費方案不支援。但輕量使用實際費用為零
- **BIM 評分為獨立維度**：通用工程人才評分（M/N/E/D 體系，門檻 20 分）和 BIM 主任職缺適配度（100 分制，5 維度）是兩套獨立評分，並排顯示
- **區域選擇 asia-east1**：部署在台灣區域以降低延遲
- **不動現有檔案**：所有新增內容嚴格放在 `web/` 子目錄，CLAUDE.md、scripts、references 等一律不修改

### 拒絕的選項（省得重議）
- **Cloud Functions gen2 (Python)**：部署大小限制 500MB，MarkItDown 全依賴可能超過。且 Cloud Run 給予更多控制（memory、timeout、Docker）
- **直接 import 原始腳本**：Cloud Run 容器獨立運行，無法存取本機路徑；且修改原始腳本違反 CLAUDE.md 約束
- **純前端方案（client-side）**：PDF 轉檔和評分需要 Python 後端，無法在瀏覽器端執行 MarkItDown
- **使用嵌入式 Python 3.14**：僅為本機 Windows 工具，Cloud Run 跑 Linux，需用標準 Docker image（python:3.12-slim）
- **Spark 免費方案**：不支援 Cloud Run / Cloud Functions，必須 Blaze
